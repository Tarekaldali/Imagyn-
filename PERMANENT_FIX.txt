# ============================================
# PERMANENT FIX FOR "FAILED TO FETCH" ERROR
# ============================================

## ROOT CAUSE:
The "Failed to fetch" error happens when the **Backend API Server** (port 8000) is NOT running.
The frontend tries to connect to http://localhost:8000 for login/register but can't reach it.

## PERMANENT SOLUTION:

### Step 1: Fix the .env file (ALREADY DONE ✅)
The .env file had invalid fields that were removed:
- ❌ Removed: R2_TOKEN
- ❌ Removed: R2_ENDPOINT
- ✅ These fields are not needed and caused the backend to crash

### Step 2: ALWAYS Start Backend First
The backend MUST be running before you try to login/register.

## HOW TO START (3 OPTIONS):

### OPTION 1: Use START.bat (Recommended)
```batch
1. Double-click: START.bat
2. Wait for 3 windows to open
3. DO NOT CLOSE any of the 3 windows
4. Go to: http://localhost:8890/login.html
```

### OPTION 2: Start Backend Only (For Testing)
```batch
1. Double-click: START_BACKEND_ONLY.bat
2. Keep window open
3. Go to: http://localhost:8890/login.html
```

### OPTION 3: Manual Start (For Developers)
```batch
# Terminal 1 - Backend API (MOST IMPORTANT!)
cd ai_platform\backend
venv\Scripts\python.exe -m uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload

# Terminal 2 - ComfyUI
.venv\Scripts\python.exe main.py --port 8189

# Terminal 3 - Flask Web
cd web_wrapper
..\.venv\Scripts\python.exe flask_server.py
```

## VERIFICATION CHECKLIST:

### ✅ Backend is Running:
```powershell
# Test 1: Check port
Test-NetConnection -ComputerName localhost -Port 8000

# Test 2: Check health endpoint
curl http://localhost:8000/health

# Expected: {"status":"healthy","timestamp":1234567890}
```

### ✅ Backend API Docs:
Open in browser: http://localhost:8000/docs
Should see FastAPI Swagger documentation

### ✅ Test Login Endpoint:
```powershell
curl http://localhost:8000/api/auth/login -Method POST -ContentType "application/json"
```

## TROUBLESHOOTING:

### Problem: Backend won't start
**Solution:**
```batch
cd ai_platform\backend
venv\Scripts\pip.exe install -r requirements.txt
```

### Problem: "Port 8000 already in use"
**Solution:**
```powershell
# Find and kill process using port 8000
netstat -ano | findstr :8000
taskkill /F /PID <PID_NUMBER>
```

### Problem: Still getting "Failed to fetch"
**Check these in order:**

1. **Is backend running?**
   ```powershell
   Test-NetConnection localhost -Port 8000
   ```
   - If FALSE: Start backend with START_BACKEND_ONLY.bat

2. **Check browser console (F12)**
   - Look for actual error message
   - Check if it's trying to connect to http://localhost:8000

3. **Check CORS**
   - Already fixed in app/main.py
   - Should see: allow_origins=["*"]

4. **Check .env file**
   - Should NOT have R2_TOKEN or R2_ENDPOINT
   - Should have SUPABASE_URL, SUPABASE_KEY, SUPABASE_SERVICE_ROLE_KEY

5. **Test with curl**
   ```powershell
   curl http://localhost:8000/health
   ```
